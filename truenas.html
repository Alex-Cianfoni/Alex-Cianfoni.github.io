Hardware / Storage / Networking

TrueNAS Build

A maximum-use NAS project built around the Gigabyte GA-EP45-DS4P, combining legacy IDE storage, six SATA devices, PCIe NVMe storage, ZFS redundancy, and upgraded networking.

Download full project guide

System plan

SYSTEM 01

Core Platform

  • Gigabyte GA-EP45-DS4P motherboard
  • Intel Core 2 Quad processor
  • About 7.5–8 GB DDR2 installed
  • Preferred memory target: 16 GB
  • TrueNAS SCALE operating system
SYSTEM 02

Primary Objective

Use the board's IDE, SATA, and PCIe expansion capabilities while separating slow archive storage, primary storage, and faster NVMe workloads into independent ZFS pools.

Storage architecture

POOL 01

ArchivePool

Two IDE/PATA hard drives configured as a ZFS mirror.

Cold storage / Backups

POOL 02

MainPool

Six motherboard SATA drives configured as three mirrors or RAIDZ2.

Primary SMB / NFS storage

POOL 03

FastPool

Four NVMe drives on a compatible PCIe adapter, arranged as two striped mirrors.

VMs / Scratch space / Fast shares

POOL 04

Auxiliary NVMe

One additional NVMe device reserved for testing as SLOG or L2ARC only after measuring the actual workload.

Experimental cache / Log role

Network design

Dual Gigabit Ethernet

The two onboard 1 GbE ports can be combined using LACP for redundancy and greater total throughput across multiple clients.

About 220–240 MB/s aggregate across multiple clients

10 Gigabit Upgrade

A supported 10 GbE adapter is the practical path to approximately 300 MB/s or more from one workstation, assuming the storage and CPU can keep up.

Higher single-client performance

Major engineering challenges

NVMe Compatibility

A passive four-drive M.2 adapter normally requires PCIe bifurcation. This older platform may need a more expensive PCIe-switch-based adapter to expose all four NVMe drives.

Boot Device Conflict

Using all six SATA ports for MainPool leaves no onboard SATA port for the TrueNAS boot SSD. The design must use a USB-attached SSD, reserve a SATA port, or add a supported storage controller.

Memory Before Cache

The guide recommends upgrading to 16 GB of RAM before adding L2ARC or using NVMe as a substitute for memory.

Cooling and Power

The full drive plan needs a quality 600–700 W power supply, sufficient connectors, direct airflow over NVMe adapters, and careful cable routing.

Build sequence

  1. Inventory and label every drive, cable, and bay.
  2. Upgrade and fully test the DDR2 memory.
  3. Service CPU cooling and improve case airflow.
  4. Install the power supply and verify connector capacity.
  5. Connect and test the mirrored IDE archive drives.
  6. Install the six SATA data drives in AHCI mode.
  7. Test the quad-NVMe adapter one drive at a time.
  8. Add the auxiliary NVMe only after the system is stable.
  9. Install TrueNAS SCALE on a dedicated boot device.
  10. Create pools, datasets, shares, snapshots, and replication jobs.

Validation goals

  • Memtest completes without errors.
  • Every drive is detected with the correct model and serial number.
  • SMART tests and ZFS scrubs complete successfully.
  • NVMe temperatures stay controlled under sustained writes.
  • One 1 GbE connection reaches roughly 110–120 MB/s.
  • Multiple clients can use the LACP link simultaneously.
  • Snapshots, replication, and test restores work correctly.
  • The TrueNAS configuration is backed up outside the server.
